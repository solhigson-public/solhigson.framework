# Clean Architecture (.NET)

## Project Naming

- `{AppName}.Domain` — entities, value objects, enums, domain events
- `{AppName}.Application` — services, DTOs, ViewModels, `ServicesWrapper`, `IRepositoryWrapper`
- `{AppName}.Infrastructure` — repositories, `AppDbContext`, `ReadOnlyAppDbContext`, EF configurations, `.generated.cs` files
- `{AppName}.Web.Ui` — controllers, Razor views, static assets, Startup/Program
- `{AppName}.Web.Hangfire` — background job dashboard
- `{AppName}.AppHost` — .NET Aspire orchestration
- `{AppName}.ServiceDefaults` — shared Aspire configuration

## What Lives Where (.NET Specifics)

- **Application**: `ServicesWrapper`, `IRepositoryWrapper` interface
- **Infrastructure**: `AppDbContext`, `ReadOnlyAppDbContext`, EF Core configurations, `.generated.cs` files (auto-generated by efcoretool)
- **Web.Ui**: ASP.NET Core controllers, Razor views

## DI Registration

- Autofac modules in Infrastructure MUST register implementations against Application interfaces
- MUST use `[DependencyInject]` attribute for auto-registration
- MUST NEVER resolve Infrastructure types directly from controllers
