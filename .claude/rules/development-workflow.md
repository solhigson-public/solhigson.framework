# Development Workflow

Every non-trivial feature MUST follow a **plan -> implement -> test** cycle. MUST read `.claude/templates/plan-spec-template.md` before writing any plan. MUST complete discussion before entering plan mode. MUST start every phase in plan mode. MUST end every phase with a clean build (zero errors). MUST keep phases small (15 steps max). MUST run `/review-phase` before committing. MUST follow `user-story-conventions.md`, `nfr-conventions.md`, `qa-conventions.md`.

For lifecycle flows, phase splitting, compliance checks, test phases, roadmap management, commit discipline, and post-phase reflection, MUST invoke the `development-workflow` skill.
