MUST NEVER create or modify files ending in `.generated.cs` — these are auto-generated by `Solhigson.Framework.efcoretool` (runs automatically post-build) and will be overwritten on every build. The efcoretool also generates a companion partial class file (e.g., `FooRepository.cs` alongside `FooRepository.generated.cs`) that is created once and never overwritten — MUST place custom code there. If efcoretool is unavailable or fails, MUST stop and ask the user — MUST NOT manually replicate generated output.
