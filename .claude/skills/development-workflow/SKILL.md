# Development Workflow Skill

Feature lifecycle flows and implementation patterns for the plan → implement → test workflow cycle. MUST follow `reference.md` for full procedures.

## When This Skill Is Invoked
- When starting a new feature, bug fix, or backfill (lifecycle flows)
- When starting a new implementation phase
- When splitting phases into sub-phases
- When running compliance checks or post-phase reflection
- When managing the project roadmap
- When writing user stories, QA test scripts, or NFRs

## Reference Files

MUST read the reference file matching the current task:

- **`reference.md`** — lifecycle flows, phase splitting, compliance checks, test phases, roadmap management, commit discipline, post-phase reflection
- **`user-story-reference.md`** — user story conventions: ID format, structure, priority definitions, file locations
- **`qa-reference.md`** — QA test script conventions: ID format, structure, test execution tiers, file locations
- **`nfr-reference.md`** — NFR conventions: ID format, structure, referencing, file locations
