# EF Core Data Layer Skill

Data layer patterns for Entity Framework Core: entity definitions, repository patterns, querying conventions, caching, partition-based archive pattern, and migration safety.

## When This Skill Is Invoked

- Creating or modifying entity definitions
- Writing repository queries or data access code
- Implementing the partition-based archive pattern
- Creating or reviewing EF Core migrations
- Setting up caching for entity data

## Key Conventions

- Entities MUST use `record` types with primary constructors
- Repositories are auto-generated by efcoretool — MUST NOT modify `.generated.cs` files
- Read queries MUST use `AsNoTracking()` and `ProjectToType<TDto>()`
- Write operations MUST go through `RepositoryWrapper.SaveChangesAsync()`
- Archive-aware reads MUST use `ArchiveService` methods

## Reference Files

MUST read the reference file matching the current task:

- **`references/entities-and-queries.md`** — entity templates, record conventions, querying rules, read/write split, repository patterns, and caching
- **`references/archive-pattern.md`** — partition-based archive pattern: entity structure (interface + base + active/archive), migration setup, ArchiveService queries, Hangfire jobs, and settings categories
- **`references/migration-safety.md`** — migration safety rules: data-preserving changes, rollback strategy, and deployment ordering
